补充

  • 先执行单行函数生成新列别名（？），列定完了再用WHERE筛选行，再使用聚合函数和GROUP BY聚合行。行都处理完了再用SELECT处理列。

  • 别名的使用。都在SELECT里平级AS出来的别名在SELECT里是不能用的，但在后面的WHERE和HAVING里可以用。

    SELECT Director,SUM(Domestic_sales + International_sales) AS SALE,COUNT() AS CNT,SUM(Domestic_sales + International_sales)/COUNT() AS ACG_SALE FROM movies JOIN Boxoffice ON Id = Movie_id WHERE CNT > 1 GROUP BY Director ORDER BY ACG_SALE LIMIT 1;

  • 笛卡尔积，自然连接，等/非等值连接，自连接。

    • 笛卡尔积（CROSS JOIN）它的作用就是可以把任意表进行连接，即使这两张表不相关，直接在条目级别全连起来。为了避免笛卡尔积， 可以在 WHERE加入有效的连接条件。

    • 等/非等值连接。说白了就是要避免笛卡尔积，指定一对列根据他们之间的关系筛选行。

    • 自连接。大多数 DBMS 使用（||）或者（+）来做拼接符，而在 MySQL 中的字符串拼接函数为concat()。像是最后生成的新列。
