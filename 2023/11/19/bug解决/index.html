2023-11-9 bug解决

2023/11/19 干掉bug pytorch

【已解决】RuntimeError: Trying to backward through the graph a second time

原因分析和解决

我在做Transformer训练的时候，需要把上一次的outputs结果拼贴到下一次的batch_x里，如果没有提前detach()掉outputs，就会报这个错误。

outputs = outputs.detach()
batch_x = torch.cat((batch_x, outputs), 2)

这样在一个epoch里就没有问题了。

epoch之间仍有问题

因为train的过程中也顺便做vali，vali函数中的内容也需要一并修改。

print("Epoch: {} cost time: {}".format(epoch + 1, time.time()-epoch_time))
train_loss = np.average(train_loss)
vali_loss = self.vali(vali_data, vali_loader, criterion)
test_loss = self.vali(test_data, test_loader, criterion)

loss值为脏数据nan/inf的问题

# 去除脏数据
if torch.any(torch.isnan(outputs)):
outputs = torch.where(torch.isnan(outputs), torch.full_like(outputs, 0), outputs)

总结

一个torch张量不能单纯看作data矩阵，还得考虑其在graph中的分布。
